Rules Setup Guide

How to Set Up Rules in Your IDE

Rules help guide AI assistants in your IDE by providing project-specific instructions, coding standards, and context.

Cursor IDE

Project Rules: Create .cursor/rules/ folder in project root. Add RULE.md files within named folders.
Global Rules: Go to Cursor Settings → Rules → User Rules (applies to all projects)
File Format: Markdown (.md) with frontmatter metadata

Example: .cursor/rules/n8n-workflow/RULE.md

Windsurf IDE

Step 1: Click the 3 dots in Cascade (top right)
Step 2: Select "Rules"
Step 3: Type workspace-rules as the rule name
Step 4: Set to "Always On"
Step 5: Paste your rules content in the editor that opens
Format: Markdown (.md)

💡 The rule will apply to your entire workspace

Antigravity IDE

Step 1: Click the 3 dots in agent chat (top right)
Step 2: Click "Customizations"
Step 3: Click "Workspace Rules"
Step 4: Enter workspace-rules as the name and hit Enter
Step 5: Set to "Always On"
Step 6: Paste your rules content inside and Save
Format: Markdown (.md)

💡 For global rules, edit ~/.gemini/GEMINI.md directly

AI Skills

What Are Skills?

Skills are knowledge files that teach AI agents how to use n8n middleware tools correctly. They provide patterns, best practices, and examples that help agents build workflows more reliably.

💡 Skills complement MCP tools — MCP tools execute commands, skills teach the agent what to do and how to do it correctly.

Setup Instructions

Antigravity IDE

Step 1: Create folder .agent/skills/ in your project root
Step 2: Copy the skill folders from the skills path below
Step 3: Skills auto-activate based on context when you chat with the agent
Format: Each skill has a folder with SKILL.md file

💡 Skills are loaded automatically when matching your queries

Cursor IDE

Step 1: Create folder .cursor/rules/ in your project root
Step 2: For each skill, create a subfolder (e.g., .cursor/rules/n8n-middleware-tools/)
Step 3: Copy the SKILL.md file and rename to RULE.md
Format: Markdown with YAML frontmatter

💡 Example: .cursor/rules/n8n-middleware-tools/RULE.md

Windsurf IDE

Step 1: Click the 3 dots in Cascade (top right)
Step 2: Select "Rules"
Step 3: Create rules named after each skill (e.g., n8n-middleware-tools)
Step 4: Paste the content from each SKILL.md file
Step 5: Set to "Always On" or "Agent Requested"

💡 "Agent Requested" activates skills only when relevant

Claude Code / Other IDEs

Option A: Copy skills to ~/.claude/skills/ (global)
Option B: Copy skills to project's .agent/skills/ folder
Format: Markdown files with YAML frontmatter
